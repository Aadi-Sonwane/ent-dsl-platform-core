# BuildOS Platform ‚Äî `project.yml` Parameters Reference Dictionary

> **Dual-audience:** Technical readers use the constraints column for validation rules;
> non-technical readers read the description column to understand what each field does.

All values go inside a file named `project.yml` at the root of your repository. The
framework validates every field against the schema at
`resources/schemas/project-schema.yml` before any build step runs.

### Scope Legend

| Scope | Meaning |
|---|---|
| **Global Only** | Set in `resources/platform/global.yml` ‚Äî the Single Source of Truth. Do NOT put in `project.yml`. |
| **Local Only** | Set per-application in `project.yml`. Required identity metadata that varies per project. |
| **Cascading Overridable** | Has a default in `global.yml` (or base framework). A `project.yml` value overrides it for that specific application. |

## 1. `project` Block ‚Äî Project Identity

### `project.metadata`

| Parameter Path | Type | Required | Default | Constraints / Regex | Non-Technical Description | Scope |
|---|---|---|---|---|---|---|
| `project.metadata.projectName` | string | **Yes** | ‚Äî | Pattern: `^[a-z0-9]([a-z0-9-]*[a-z0-9])?$` | A short, lowercase name for your project. Used to create folder names in Jenkins, repository IDs in Nexus, and artifact prefixes. Hyphens allowed. Examples: `my-java-app`, `payment-service`. | **Local Only** |
| `project.metadata.businessUnit` | string | **Yes** | ‚Äî | Pattern: `^[A-Z][a-zA-Z0-9-]{2,32}$` | Your division or department name within the company. Starts with a capital letter. Examples: `PlatformEngineering`, `Payments`, `DataPlatform`. | **Local Only** |
| `project.metadata.team` | string | **Yes** | ‚Äî | Pattern: `^[a-z0-9]([a-z0-9-]*[a-z0-9])?$` | The team that owns this project. Lowercase, hyphen-separated. Examples: `core-devops`, `backend-api`. | **Local Only** |
| `project.metadata.repositoryUrl` | string | **Yes** | ‚Äî | Pattern: `^https?://.*` | The full URL to your source code repository (starting with http:// or https://). Jenkins clones code from here. Example: `http://buildos-git:3000/gitea_admin/demo-java-service.git`. | **Local Only** |
| `project.metadata.scmCredentialsId` | string | No | `git` | ‚Äî | The Jenkins credential ID used to clone the repository. Our platform standard is `git`. Change only if you use a different credentials store. | **Local Only** |
| `project.metadata.description` | string | No | ‚Äî | Max length: 1024 chars | A sentence or two explaining what your project does. Displayed in Jenkins folder descriptions and dashboards. | **Local Only** |
| `project.metadata.tags` | list of string | No | ‚Äî | ‚Äî | Labels to help search and categorize your project in Jenkins views. Examples: `spring-boot`, `microservice`, `critical`. | **Local Only** |

### `project.version`

| Parameter Path | Type | Required | Default | Constraints / Regex | Non-Technical Description | Scope |
|---|---|---|---|---|---|---|
| `project.version` | string | No | `1.0.0-SNAPSHOT` | Pattern: `^\d+\.\d+\.\d+(-SNAPSHOT)?$` | The version number of your project. Follows semantic versioning (major.minor.patch). Add `-SNAPSHOT` for in-development builds. Example: `2.1.0-SNAPSHOT`. | **Local Only** |

### `project.parent`

| Parameter Path | Type | Required | Default | Constraints / Regex | Non-Technical Description | Scope |
|---|---|---|---|---|---|---|
| `project.parent.ref` | string | **Yes** (if parent block present) | ‚Äî | Pattern: `^[a-z0-9]([a-z0-9-]*[a-z0-9])?$` | The project name of a parent project whose settings you want to reuse. The parent's configuration is loaded first, then your overrides are merged on top. | **Local Only** |
| `project.parent.inherit` | list of string | No | ‚Äî | Allowed: `nexus`, `security`, `quality`, `maven`, `branchGovernance`, `rbac`, `policies`, `observability` | Which sections of the parent project's configuration to inherit. Pick only the sections you want to copy. If empty, nothing is inherited from the parent even though the parent is referenced. | **Local Only** |

## Quick-Reference: Required vs. Optional by Section

| Section | Required fields | Everything else |
|---|---|---|
| `project` | `projectName`, `businessUnit`, `team`, `repositoryUrl` | All optional with defaults |
| `nexus` | `url` (if section present) | All optional with defaults |
| `branchGovernance` | None | All optional |
| `rbac` | `roles[].name`, `roles[].permissions` (if section present) | All optional |
| `maven` | `snapshotRepository.id/url` + `releaseRepository.id/url` (if distributionManagement present) | All optional with defaults |
| `security` | None | All optional with defaults |
| `quality` | None | All optional with defaults |
| `observability` | None | All optional with defaults |
